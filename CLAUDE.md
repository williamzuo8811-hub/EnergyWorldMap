# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## What this is

An interactive **world map of global energy projects** (е…ЁзҗғиғҪжәҗйЎ№зӣ®дё–з•Ңең°еӣҫ). A zoomable Leaflet
tile map plots ~3,900 energy projects worldwide, filterable by category, region, status, and year,
with live KPIs, a category bar chart, a top-projects list, project detail cards, an investment
heatmap mode, and "рҹҶ• last 12 months" highlighting. The UI is in Chinese.

**Pure frontend, zero build.** Plain `HTML + CSS + vanilla JS`, no package manager, no bundler,
no transpile step. Everything (map library + data) is local except map tiles, which load from
Esri/OSM over the network.

**Installable PWA + offline.** `manifest.json` + `sw.js` make the site installable and make repeat
visits instant / offline (everything but tiles is cached). `sw.js` uses stale-while-revalidate for
same-origin assets (cache key strips the deploy's `?v=<sha>` query so the cache stays bounded),
network-first for navigations, and passes tiles through (no caching of huge tile sets). All `<script>`
tags carry `defer` so the page shell paints before the ~3 MB of data executes вҖ” **scripts that parse
`index.html`'s load order must tolerate `<script defer src=вҖҰ>` (regex `/<script[^>]*\ssrc="вҖҰ"/`).**
The social card `assets/og-cover.png` and PWA icons (`assets/icon-{192,512}.png`) are generated by
`node scripts/build-og.js` (pure-Node PNG encoder; the OG `<meta>` must point at the **PNG**, not the
SVG вҖ” social platforms don't render SVG `og:image`). Re-run `build-og.js` when the project count or
brand changes; the count is read live from the data layer.

## Running & verifying

There is no build, test, or lint tooling. To see changes:

- Simplest: open `index.html` directly in a browser (Chrome/Edge recommended).
- Local server (optional, Windows/PowerShell): `powershell -ExecutionPolicy Bypass -File serve.ps1 -Port 4173`, then open `http://localhost:4173/`.
- On Linux/macOS any static server works, e.g. `python3 -m http.server 4173`.

Map tiles require network access; project markers and all logic work offline. After editing,
verify in a browser вҖ” there is no build step.

## Architecture

Two layers loaded as ordinary `<script>` tags in `index.html` (order is load-bearing):

1. **Vendored libraries** (`lib/`): `leaflet.js`, `leaflet.markercluster.js`, `leaflet-heat.js`
   plus their CSS and marker images. Do not edit these.
2. **Data files** (`js/data*.js`, `js/progress.js`): each attaches data to globals on `window`.
3. **`js/util.js`**: DOM-free pure logic shared by `app.js`, `globe.js` and the unit tests вҖ”
   `SUB_DEFS` + `classifySub`, `parseCapacity`, the WGS-84вҶ’GCJ-02 conversion (`wgs2gcj`/`outOfChina`),
   `normalizeOwner`, plus the de-duplicated cross-page helpers: `LABELS_EN` (cat/region/status zhвҶ’en
   maps), `invMagnitude(n, lang)` (money magnitude without currency sign вҖ” callers prepend `вүҲ$`/`$`),
   `capFmtMW(mw)`, and `buildProjects(ENERGY, EXTRA, PROGRESS, {requireCoord})` (the dedup-by-name +
   progress + `classifySub` + `parseCapacity` assembly used by **both** `app.js` and `globe.js`; the
   3D globe passes `requireCoord:true`). Dual-export: attaches `window.ENERGY_UTIL` in the browser,
   `module.exports` under Node (so `scripts/test-units.js` can `require` it). Must load **before** `app.js`.
4. **`js/app.js`**: the application/UI logic вҖ” map setup, basemap switching, filtering,
   stats, detail cards, heatmap, mobile drawers. Runs inside one IIFE; pulls the pure helpers
   from `window.ENERGY_UTIL`.

### 3D globe view (`globe.html` + `js/globe.js`)

A second, standalone page renders the same dataset as a **three-dimensional globe** (the 2D map links to it
via the `рҹҢҗ 3D ең°зҗғ` button in `index.html`'s map-tools; the globe links back via `рҹ—әпёҸ е№ійқўең°еӣҫ`). It is the
"visual ceiling" view and is fully independent of `app.js` вҖ” but **reuses the exact data layer**: `globe.html`
loads the same `js/data*.js` + `js/progress.js` + `js/util.js` script order, then `js/region-insights.js`,
`lib/world-110m.js`, `lib/globe.gl.min.js`, and `js/globe.js`. `globe.js` rebuilds `PROJECTS` with the same dedup-by-name / progress /
`classifySub` / `parseCapacity` pipeline as `app.js` (it keeps only entries with a `coord`). Three visual encodings:
**extruded columns** (`pointsData`; bar height вҲқ `вҲҡweight` normalized to the global max, color = category, white when
focused вҖ” `weightVal` switches investment вҮ„ `capMW` via the жҹұй«ҳ toggle), **cross-border arcs** (`arcsData`; every
consecutive segment of a project's `route` becomes a great-circle arc with an animated dashed gradient, stroke вҲқ `inv`),
and a **hexed dark globe** (`hexPolygonsData` dotted land from the vendored Natural-Earth-110m GeoJSON, plus graticules
and an atmosphere glow over a CSS starfield). Left-panel chips filter by category/region/status/year-preset (same
client-exclusion rule in the investment KPI as `app.js`); clicking a column opens a detail card and flies the camera
(`pointOfView`) to it with a pulse ring; auto-rotate via `controls()`. The left panel's `рҹҢҚ еҢәеҹҹиғҪжәҗжҙһеҜҹ` button
opens a region-insight card (`showRegionInsight`, same `window.REGION_INSIGHT` source as the 2D map) вҖ” summary +
KPIs + иғҪжәҗ/з”өзҪ‘/жңҖж–°еҠЁжҖҒ/жңәдјҡ/йҮҚзӮ№еӣҪе®¶, в—Ӯв–ё region nav, and `рҹҺҜ йЈһеҲ°иҜҘеҢә` flies the camera to the region centroid
(`REGION_CENTROID`, mean of the region's project coords); it shares the detail card's slot (mutually exclusive).
A debug handle is exposed at `window.__GLOBE__`.
The library `lib/globe.gl.min.js` is a vendored standalone UMD bundle (globe.gl + three.js, global `Globe`) and
`lib/world-110m.js` a precomputed GeoJSON global (`window.WORLD_GEO`) вҖ” **do not edit either**; everything is offline
(no CDN, no build), like the rest of the project.

### Subcategory auto-classification (`SUB_DEFS` in `js/util.js`)

Subcategories are **not stored** on projects вҖ” they are derived. `SUB_DEFS` maps each category to
an *ordered* list of subcategory rules matched against the project's `name/en/cap/desc` (some use a
custom `fn` matching `owner`, `region`, etc.). `classifySub` returns the first matching rule's key;
the **last entry in each list has no rule and is the catch-all bucket**. Order matters вҖ” more
specific rules must come before broader ones. New projects are categorized automatically with no
manual tagging. To change subcategories, edit `SUB_DEFS` in `js/util.js` (labels, `zh`/`en`
keywords, or order); `scripts/test-units.js` has assertions guarding the matcher order.

### Coordinate handling

Project `coord` is always `[lng, lat]` in **WGS-84**. `toLatLng()` returns Leaflet's `[lat, lng]`.
`js/util.js` contains a full WGS-84вҶ’GCJ-02 conversion (`wgs2gcj`, applied only when a basemap's
`crs` is `gcj02`). The two active basemaps (`dark` Esri, `osm`) are both WGS-84, so no shift is
currently applied вҖ” but keep coordinates in WGS-84 so the conversion stays correct if a Chinese
basemap is re-added.

### Categories drive the UI

`CATEGORIES` in `data.js` (key вҶ’ `{name, short, color, icon}`) is the single source for the legend,
filters, marker colors, cluster colors, and stat bars. Adding a category there makes it flow through
the whole UI automatically вҖ” but you must also add a matching `SUB_DEFS[<key>]` entry in `js/util.js`,
or that category's projects get no subcategory chips.

### Derived capacity & metrics

`parseCapacity(cap)` (in `js/util.js`) turns the free-text `cap` into structured numbers attached to each

(storage energy), `capKm` (line/route length), `capKbd` (oil дёҮжЎ¶/ж—Ҙ), `capWty` (mass дёҮеҗЁ/е№ҙ), `capPF` (AI compute
in PFLOPS: `EFLOPS/exaflops`Г—1000, `дёҮP/дёҮPFlops`Г—10000, `PFLOPS` and bare `ж•°еӯ—P` like `7209P` Г—1),
`capPax` (passenger throughput дёҮдәәж¬Ў/е№ҙ, дәҝдәәж¬ЎГ—10000), `capTeu` (container throughput дёҮTEU/дёҮж Үз®ұ),

unit, sums `A+B` lists, multiplies `NГ—M`, and uses lookaheads so `MW`/`GW` don't swallow `MWh`/`GWh` (and bare

## Project data conventions

A project object (see `data.js` header comment for the full field reference):

```js
{
  id, name, en, country,
  region,            // one of REGIONS: дёңдәҡ/дёңеҚ—дәҡ/еҚ—дәҡ/дёӯдәҡ/дёӯдёң/ж¬§жҙІ/еҢ—зҫҺ/еҚ—зҫҺ/йқһжҙІ/еӨ§жҙӢжҙІпјҲ'дёӯеӣҪ' дёәеҺҶеҸІеҖјпјҢutil.buildProjects еҠ иҪҪж—¶е№¶е…Ҙ 'дёңдәҡ'пјү
  cat,               // one of CATEGORIES keys: renewable/grid/storage/ci/datacenter/transport/petro/mining/client
  coord: [lng, lat], // WGS-84
  cap, inv, invText, // inv is a number in дәҝзҫҺе…ғ (100M USD) used for sizing/stats; invText is display string
  status,            // и§„еҲ’ / еңЁе»ә / жҠ•иҝҗ
  year,              // milestone year (drives the year slider)
  updated,           // 'YYYY-MM'; >= META.recentSince вҮ’ flagged рҹҶ• (near-term)
  owner, flagship,   // flagship:true adds a pulsing marker
  desc, detail,      // desc = short tooltip/list text; detail = full paragraph in the detail card
  route,             // optional [[lng,lat], ...] connector line
}
```

- **Inclusion threshold**: projects with investment > 50M RMB (~$7M).
- **New project**: copy an entry, edit fields, set `updated` to the current month вҶ’ it appears and is auto-flagged рҹҶ•.
- **Changed project**: update `status`/`cap`/`inv`/`desc`/`detail` and bump `updated`.
- **"Recent" window**: controlled by `META.recentSince` (currently `'2026-01'`); `META.lastUpdated`
  (`'2026-06'`) is shown in the header. Bump both when refreshing the dataset.
- The category key `client` (еӣҪйҷ…еӨ§е®ўжҲ·) groups overseas projects of 54 Chinese companies by `owner`
  (see the `SUB_DEFS.client` `fn` matchers): the original 14 (`data-clients.js`, id 1570вҖ“1698) plus the 40
  from theгҖҠзү№й”җеҫ· 50 е®¶дёӯеӣҪеҮәжө·еӨ§е®ўжҲ·гҖӢlist plus deeper passes (`data-clients2.js`, id 4000вҖ“4522).

## Data-refresh workflow

The maintainer triggers refreshes by asking (in Chinese) "еҲ·ж–°иғҪжәҗең°еӣҫж•°жҚ®" / "еҲ·ж–°жңҖж–°иҝӣеұ•". The

add new projects (with full `detail`), update changed projects' `status`/`cap`/`inv`/`updated`,
extend `js/progress.js` by `id`, and set `META.lastUpdated` to the current month. All content is in
Simplified Chinese, UTF-8 вҖ” match the existing tone and field style. A new regional batch goes in its
own `js/data-*.js` with a distinct id range (append via `.concat`, register in `index.html`). Always
finish by running `node scripts/validate-data.js` and confirming a clean exit before committing.
