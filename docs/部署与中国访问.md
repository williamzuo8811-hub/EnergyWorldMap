# 中国大陆访问 & Cloudflare Pages 部署指南

> 痛点：`https://williamzuo8811-hub.github.io/EnergyWorldMap/` 在中国大陆经常打不开。
> 根因不在代码，而在托管层——`github.io` 域名被 GFW 干扰。本文给出可落地的解决办法。

## 一图看懂问题分两层

| 层 | 现状 | 国内表现 |
|---|---|---|
| **托管**（站点 HTML/JS/数据） | GitHub Pages（`*.github.io`） | ❌ 被墙/限速，"打不开"的真凶 |
| **地图瓦片**（底图图片） | 已内置高德底图（国内服务器、GCJ-02 纠偏） | ✅ 已为国内访客自动默认高德（见下） |

> 站点是**完全可搬家**的：无 CNAME、PWA `scope/start_url` 均为相对路径 `./`、代码无写死的 `github.io`
> 绝对地址。换托管 = 把文件原样部署到新平台，**一行 URL 都不用改**。

---

## 一、地图瓦片（已自动处理）

`js/app.js` 在初始化时用 `preferAmap()` 按**浏览器时区**识别是否身处中国大陆：

- 时区为 `Asia/Shanghai`（含 `Chongqing/Urumqi/Harbin/PRC`，均为 UTC+8）→ 视为大陆。

→ 默认底图自动切到**高德（中国）**（国内服务器、加载快）；海外访客仍默认 **OSM（国际）**。
**只用时区、不用浏览器语言**判断：海外华人常把语言设为 `zh-CN`，若按语言会被误判（例如澳洲访客一打开就看到高德）。
任何访客都可在地图右上角的底图切换里手动改选。**无需任何配置，部署后即生效。**

## 二、托管方案：Cloudflare Pages（推荐，无需备案）

免费、自动从 GitHub 仓库部署、绑定自定义域名后国内可访问（稳定性远胜 `github.io`）。

### 步骤

1. **登录** [dash.cloudflare.com](https://dash.cloudflare.com)（没有账号先免费注册）。
2. 左侧 **Workers & Pages** → **Create** → 选 **Pages** 选项卡 → **Connect to Git**。
3. 授权并选择仓库 **`williamzuo8811-hub/energyworldmap`**。
4. **构建设置（关键——这是零构建静态站，别填错）**：

   | 字段 | 值 |
   |---|---|
   | Production branch | `main` |
   | Framework preset | **None** |
   | Build command | **留空** |
   | Build output directory | `/`（仓库根目录即站点） |
   | Root directory | `/` |

5. **Save and Deploy**。约 1 分钟后得到一个 `xxxx.pages.dev` 的地址，全球（含海外）即时可用。
6. 之后每次推送到 `main`，Cloudflare 会**自动重新部署**（和 GitHub Pages 体验一致）。

### 绑定自定义域名（强烈建议，提升国内访问）

裸的 `*.pages.dev` 在国内仍可能波动；**绑自定义域名**更稳：

1. 你需要一个域名（在任意注册商买，几十元/年）。
2. 把该域名的 DNS 托管到 Cloudflare（注册商处把 NS 改成 Cloudflare 给的两条）。
3. Pages 项目 → **Custom domains** → **Set up a custom domain** → 输入你的域名 → 按提示添加 CNAME。
4. 等证书签发（几分钟）即可用 `https://你的域名/` 访问。

> ⚠️ **诚实提示**：Cloudflare **免费版不走中国大陆境内节点**（境内节点属企业版 China Network）。
> 所以 Cloudflare Pages 的定位是"**比 github.io 好很多、能稳定打开**"，但不是"大陆秒开"。
> 若要"境内秒开 + 长期稳定"，最终方案是 **国内云对象存储 + CDN + ICP 备案**（见下）。

---

### 仓库内的 `wrangler.jsonc`（Cloudflare Workers Builds 路径）

若仓库接入的是 **Cloudflare Workers（Workers Builds）** 而非经典 Pages，其 CI 会执行
`npx wrangler deploy`，需要仓库根有 `wrangler.jsonc` 才能成功（否则一启动即失败）。本仓库
已内置该文件：把整个根目录作为 **Workers 静态资源（assets-only，无 Worker 脚本）** 上传，
仍是零构建；`.assetsignore` 排除 `scripts/`、`docs/`、`*.md` 等非站点文件。无需 `package.json`，
部署命令保持 Cloudflare 默认的 `npx wrangler deploy` 即可。经典 Cloudflare Pages 路径（上文
步骤）与 GitHub Pages 都不读取该文件，互不影响。

## 三、备选：国内云对象存储 + CDN（最快，但需 ICP 备案）

适合追求"大陆境内秒开"。代表：阿里云 OSS、腾讯云 COS、七牛云 Kodo（+ 各家 CDN）。

- **前提：ICP 备案**——需要一个域名 + 国内主体/个人身份证，**审核约 1–3 周，免费**。
  （未备案的大陆 IP 会被运营商封 80/443 端口，所以这步绕不开。）
- 备案通过后：把整个仓库文件上传到 Bucket → 开启静态网站托管 → 套 CDN 加速 → 绑定已备案域名。
- 成本：纯静态小站，通常每月几块钱甚至在免费额度内。

需要时我可以给你写**逐步上传 + CDN 配置**的细则。

## 四、不推荐的方案（避坑）

- **Gitee Pages**：曾是国内 GitHub Pages 镜像，但免费 Pages 服务自 2021 起已暂停/需人工审核，基本不可用。
- **Vercel / Netlify**：在国内同样经常被墙，换汤不换药。

## 五、访问统计（可选，5 分钟启用）

三个页面（`index.html` / `globe.html` / `coach.html`）的 `</body>` 前都预置了一段**注释掉的**
Cloudflare Web Analytics 代码。它免费、无 Cookie、不采集个人数据（符合多数隐私法规），能回答：
多少人在用、用哪个页面（地图/地球/陪练）、访客来自哪些国家、设备与来源渠道。

启用步骤：

1. 登录 [Cloudflare 仪表盘](https://dash.cloudflare.com) → 左侧 **Analytics & Logs → Web Analytics**；
2. **Add a site**，填入站点域名（GitHub Pages 域名也可以——Web Analytics 不要求域名托管在 Cloudflare）；
3. 选择"手动安装"，复制弹出代码里 `token` 的值；
4. 在三个 HTML 文件里搜索 `YOUR_CF_TOKEN`，替换为你的 token，并**去掉外层注释**（`<!--` 与 `-->`）；
5. 提交部署后约几分钟，仪表盘即可看到数据。

注意：统计脚本 `static.cloudflareinsights.com` 在中国大陆可能被拦截——**不影响页面本身**
（`defer` 加载失败静默），只是大陆访客可能不计入统计。若主要受众在大陆且必须统计，
可换 51.la / 百度统计等国内方案（同样是一段 `<script>`，但注意它们通常使用 Cookie）。

## 速查：换托管后要不要改东西？

- **不用改 URL**：站点全相对路径，搬到任何平台都能跑。
- **PWA / Service Worker**：新域名首访会重新注册，`sw.js` 缓存键已剥离 `?v=` 查询，正常工作。
- **缓存击穿**：GitHub Actions 的 `?v=<sha>` 注入只在 GitHub Pages 部署时生效；Cloudflare Pages
  每次部署会自动刷新自家边缘缓存，普通刷新即见最新，无需额外处理。
